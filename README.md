## F. Project Scenario
StudySmart AI simulates a smart academic planning application. The system receives a set of
study tasks and helps the student decide which tasks should be prioritized and which planning
strategy is most suitable. Each task has  attributes such as study time, importance, deadline, difficulty, and task type.
The project simulates a real-world software development situation: imagine you are designing a  smart  study-planning  application  that  automatically  recommends  the  best  algorithmic
strategy based on task characteristics and student constraints.
## G. Project Goal
The main goal of this project is to create a program that can:
- Generate or input different study-task scenarios.
- Implement multiple algorithmic strategies for task planning.
- Measure the performance and quality of each strategy.
- Extract features from the task scenario.
- Use an AI/ML model to predict the most suitable strategy.
- Compare the AI-predicted strategy with the actual best strategy.
- Present the results clearly using structured tables.

## I. Group Component: System Development Task (57%)
Each group must design and implement the StudySmart AI system using C/C++. The system
should contain the following required modules.
### Module 1: Task Scenario Generation / Input
The system must allow Groups should prepare different scenarios, such as:
- Low-pressure scenario: available study time is enough for most tasks.
- High-pressure scenario: available study time is much lower than total required time.
- Deadline-focused scenario: several tasks have urgent deadlines.
- Importance-focused scenario: some tasks have much higher importance scores than others.
	Each group must test **at least two different study-task scenarios** against all three algorithmic strategies (Modules 2, 3, and 4). The results of each scenario must be recorded in the performance comparison table (Module 6).
### Module 2: Sorting / Divide-and-Conquer Module

in the report) to rank study tasks based on one selected criterion, such as deadline, importance, difficulty, study time, urgency, or an importance-to-time ratio.

## Module 3: Greedy Planning Module
Students must implement one greedy strategy to create a quick study plan. Possible greedy
rules include highest importance first, earliest deadline first, shortest study time first, or
highest importance-to-time ratio first. The report must explain the selected rule and discuss
whether it always gives the optimal solution.
## Module 4: Dynamic Programming Module
Students must implement the 0/1 Knapsack algorithm using Dynamic Programming. In this
project, the available study time is the capacity. Each task has a required study time and an importance score. The DP algorithm should select the combination of tasks that maximizes
total importance without exceeding the available study time.

| Selected Task              | Study Time | Importance Score |
| -------------------------- | ---------- | ---------------- |
| Revise Dynamic Programming | 3          | 5                |
| Practise Graph Algorithms  | 4          | 6                |
| Review Sorting Algorithms  | 2          | 3                |
| **TOTAL**                      | **9**          | **14**               |
> Table 4: Example of Tasks Selected by Dynamic Programming Module

**Optional  Extension**: Backtracking Approach: Backtracking is not required for the group implementation. However, students who choose  Backtracking  as their individual topic may
explain and apply it to a small version of the study-task selection problem. For example, they
may show how Backtracking explores possible task subsets and selects the best combination
without  exceeding  the  available  study  time.  Since  Backtracking  has  exponential  time
complexity, it should only be demonstrated using a small number of tasks.
### Module 5: AI/ML Recommendation Module
Each  group  must  include  a  simple  AI/ML  module  that  predicts  the  most  suitable  planning strategy  for  a  given  study-task  scenario. The  AI/ML  module  may  be implemented  using  a simple Decision  Tree, k-Nearest  Neighbours  (k-NN), or  a  manually  designed  rule-based decision tree. The model must be trained or built using a minimum of 8 to 15 labelled scenario examples, where each example consists of the extracted features listed in Table 5 and a known best strategy label. Students must include these training examples in the report to demonstrate how the model was built.
> **Note**: The AI/ML component may be lightweight; students are not required to use advanced
ML libraries, but they must clearly define the input features, prediction logic, and strategy classes.


The AI/ML module should use scenario features such as:

| Feature                   | Description                                                  |
| ------------------------- | ------------------------------------------------------------ |
| Number of tasks           | Total number of study tasks in the scenario                  |
| Total required study time | Sum of all task durations                                    |
| Available study time      | Total study time available to the student                    |
| Time pressure ratio       | Total required time divided by available time                |
| Average importance        | Average importance score of all tasks                        |
| Deadline tightness        | How close the deadlines are                                  |
| Importance variation      | Whether task importance scores are similar or very different |
> Table 5: Input Features for the AI/ML Recommendation Module

The model should predict one of the following strategy classes:

| Prediction Class      | Meaning                                                                      |
| --------------------- | ---------------------------------------------------------------------------- |
| Sorting-based ranking | Suitable when the main goal is prioritization.                               |
| Greedy strategy       | Suitable when a fast, simple plan is needed.                                 |
| Dynamic Programming   | Suitable when study time is limited and optimal task selection is important. |
> Table 6: AI/ML Strategy Prediction Classes
